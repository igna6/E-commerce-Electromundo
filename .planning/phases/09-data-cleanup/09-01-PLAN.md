       - Handles edge cases: empty strings, single characters, already mixed case
       - Example: "BICICLETA GCA MOUNTAINBIKE RODADO 29" → "Bicicleta Gca Mountainbike Rodado 29"

       - `CartSummary.tsx` or cart item components — product name in cart
       - Search results, order summaries, breadcrumbs, page titles
       - Use grep to find any other `product.name` or `item.name` renders across `web/src/`

<task type="auto">
  <name>Task 2: Extend CSV import to handle optional price column</name>
  <files>server/src/services/admin/importProducts.service.ts</files>
  <action>
    1. In `importProducts.service.ts`, add price column detection alongside existing SKU/name/stock detection:
       - Add `priceIdx` using header matching for: "precio", "price", "valor", "costo", "pvp", "importe" (same normalize pattern as other headers)
       - Price column is OPTIONAL — do not throw if missing

    2. In the data row parsing loop, parse price when column exists:
       - Parse as float (supports "1234.56" or "1234,56" Argentine decimal format — replace comma with dot before parseFloat)
       - Convert to cents: `Math.round(parsedPrice * 100)`
       - If price is invalid/NaN/negative for a row, skip it (don't set price for that row, log to errors array)
       - Add `price` as optional field to the `validProducts` array type

    3. In the transaction upsert logic:
       - For NEW products (insert): use parsed price if available, otherwise keep `price: 0` as fallback
       - For EXISTING products (update): if price is available and > 0 in the CSV, update it alongside name and stock. Do NOT overwrite an existing non-zero price with 0 (only update price if CSV provides a valid positive price)

    4. Update the return value to include price info: add `priceUpdated: number` counter to track how many products got price updates.

    Do NOT change the CSV parsing function itself — it already handles commas and semicolons.
    Do NOT make price a required column — backwards compatibility with existing CSVs that have no price column.
  </action>
  <verify>
    - `cd server && npx tsc --noEmit` passes with no errors
    - The import still works with CSVs that have no price column (backwards compatible)
  </verify>
  <done>Import service accepts optional price column from CSV. New products get CSV price (or 0 fallback). Existing products get price updated only when CSV provides a valid positive price. TypeScript compiles clean.</done>
</task>
