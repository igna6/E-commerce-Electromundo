<task type="auto">
  <name>Task 2: Assign products to categories by keyword matching</name>
  <files>server/src/scripts/assignProductCategories.ts, server/package.json</files>
  <action>
    1. Create `server/src/scripts/assignProductCategories.ts` — a standalone script that:
       - Reads all products and all categories from DB
       - Matches each product to a category based on keywords in the product name
       - Updates the product's `category` FK

    2. Keyword matching logic:
       - Build a mapping of keywords → category ID (e.g., "bicicleta" → Bicicletas, "notebook" → Notebooks, "celular"/"telefono"/"smartphone" → Celulares)
       - Match against lowercased product name
       - Try most specific keywords first (subcategories), then broader ones (parent categories)
       - If no keyword matches, assign to a "General" or "Otros" catch-all category
       - Log unmatched products for manual review

    3. The script should:
       - Only update products that currently have NO category (null) — don't overwrite manual assignments
       - Be idempotent (safe to re-run)
       - Print summary: X products assigned, Y already had categories, Z unmatched
       - Be runnable via `npx tsx src/scripts/assignProductCategories.ts`

    4. Add package.json script: `"seed:assign-categories": "tsx src/scripts/assignProductCategories.ts"`

    Do NOT use AI/fuzzy matching — simple keyword includes is sufficient.
    Do NOT modify product data beyond the category FK.
  </action>
  <verify>
    - `cd server && npx tsc --noEmit` passes
    - `cd server && npm run seed:assign-categories` runs successfully
    - Query DB to confirm products have category assignments
    - Check that most products (~80%+) got assigned (some unmatched is acceptable)
  </verify>
  <done>All (or nearly all) products assigned to appropriate categories. Script logs unmatched products. Idempotent and safe to re-run. Phase 10 complete.</done>
