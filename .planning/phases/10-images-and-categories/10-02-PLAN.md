Output: Category table supports parent-child relationships, API returns nested categories, admin can create/edit subcategories.
</objective>

    1. In `server/src/entities/productCategories.ts`:
       - Add `parentCategoryId: integer().references(() => productCategoriesTable.id)` — nullable self-referencing FK
       - This allows categories to have a parent (subcategories) while top-level categories have null parentCategoryId

    2. Generate and push migration:
       - Run `cd server && npx drizzle-kit generate` to create migration file
       - Run `cd server && npx drizzle-kit push` to push schema to DB

    3. In `server/src/validators/category.ts`:
       - Add `parentCategoryId: z.number().int().positive().nullable().optional()` to both create and update schemas
       - Use `parentCategoryId?: number | null | undefined` pattern for strict mode compatibility

    4. In `web/src/types/category.ts`:
       - Add `parentCategoryId: number | null` to Category type

    Do NOT add circular reference validation yet — keep it simple (admin won't accidentally create circular references with the dropdown UI).
  </action>
  <verify>

  <action>
    **Server side:**

    1. In `categories.service.ts`:
       - `listCategories()`: Return all categories with `parentCategoryId` included in the select. Keep returning a flat list — let the frontend handle nesting if needed.
       - `createCategory(data)`: Accept `parentCategoryId`. If provided, verify the parent category exists and is not deleted before inserting.
       - `updateCategory(id, data)`: Accept `parentCategoryId`. Validate parent exists if provided. Prevent setting parentCategoryId to own id (basic circular reference check).

    2. In `categories.controller.ts`: Pass `parentCategoryId` from validated request body to service methods.

    **Frontend side:**

    3. In admin `CategoriesPage.tsx`:
       - Add a "Categoría padre" (Parent category) dropdown to the category create/edit form
       - Dropdown options: "Ninguna (categoría principal)" + all existing categories (filtered to exclude the category being edited, to prevent self-reference)
       - Display categories in the list with visual hierarchy: indent subcategories or show parent name as a prefix (e.g., "Electrónica > Celulares")
       - Sort: parent categories first, then subcategories grouped under their parent

    4. In frontend `categories.service.ts`: Ensure create/update API calls pass `parentCategoryId` in the request body.

    Keep the admin UI simple — no drag-and-drop tree, just a dropdown for parent selection and indented list display.
  </action>
